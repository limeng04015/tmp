# Claude Coding Rules (Project Standard)
1. Write minimal, clean, direct code.
2. No redundant comments, no unnecessary variables, no over-engineering.
3. Only implement what is required; do not add extra features.
4. Prioritize readability and brevity.
5. Use consistent naming and structure.
6. Modify only the parts that need changing; avoid full rewrites.
7. Output only code unless asked to explain.
8. Keep functions short and single-purpose.
